Git-Meta

https://github.com/twosigma/git-meta

Project Description

Over the summer 2022, I got the chance to participate in the 6-week Two Sigma College Mentor Connect Program. This program aimed at helping students from underrepresented groups in tech to gain experience in the tech industry by pairing them with Software Engineer mentors from Two Sigma. I was paired with a mentor from the Two Sigma SDLC Team, and got the chance to work on the Git-Meta project.

To start off with, a mono-repo can be defined as a repository that contains multiple projects all under one shared umbrella. This is useful for maintaining multiple projects that are related to each other within one codebase allowing for easier access and collaboration within a larger organization such as Two Sigma. I mostly started off with learning basic Git commands and how to use them along with understanding how pull requests and code reviews worked. All of this allowed me to get a grasp of how to start contributing to the project by dabbling on some documentation.

The changes I ended up implementing were a fex fixes to the overall documentation regarding the Git-Meta project. I also fixed two of the Git-Meta commands including the git-meta cherry pick and git-meta rebase commands. In particular, these commands were having unclear documnetation when faced with error messages and as such I fixed them so that there were more clear messages being displayed. All in all, I had a great time and learned a lot about the general workflow of contributing to an open source project.
